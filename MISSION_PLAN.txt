============================================================================
🛡️ PROJECT: MCPVANGUARD (THE VANGUARD NETWORK)
============================================================================
GOAL: A Decentralized, Real-time "AI Antivirus" and Security Middleware for 
Model Context Protocol (MCP) tool calls. Inspired by Bittensor Subnet 61.

DATE: February 21, 2026
STATUS: Architecture Phase (v1.0 Alpha)

----------------------------------------------------------------------------
1. THE CORE ARCHITECTURE: TRANSPARENT JSON-RPC PROXY
----------------------------------------------------------------------------
McpVanguard acts as a middle-man between an AI Agent (Client) and an 
MCP Server (Tool).

FLOW:
AI Agent <--> [ McpVanguard Proxy ] <--> Real MCP Server (Subprocess)

TECHNICAL MECHANISM:
- Standard Input (stdin) Interception: Capture JSON-RPC calls from Agent.
- Standard Output (stdout) Interception: Capture tool results from Server.
- Asynchronous I/O: Must maintain <10ms latency to avoid Agent timeouts.
- State Machine: Tracks the "Session State" to detect multi-turn attacks.

----------------------------------------------------------------------------
2. THE THREE LAYERS OF DEFENSE
----------------------------------------------------------------------------
LAYER 1: STATIC SIGNATURES (THE FIREWALL)
- Regex-based filtering of known malicious strings (jailbreaks).
- Path Restriction: Block access to /etc/, ~/.ssh/, and system files.
- Command Blacklist: Block 'rm -rf', 'curl | bash', etc.

LAYER 2: SEMANTIC INSPECTION (THE BRAIN)
- Small Local LLM (e.g., Llama-3-1B) scores the "Intent" of each tool call.
- Intent Risk Score (0.0 to 1.0).
- Action: Block calls with a risk score > 0.8.

LAYER 3: BEHAVIORAL ANALYSIS (THE SENTRY)
- Tracks frequency and volume of tool calls.
- Detects "Data Scraping" patterns (e.g., 100 read_file calls in 10s).
- Detects "Privilege Escalation" patterns.

----------------------------------------------------------------------------
3. THE "HUNTER" NETWORK (SUBNET 61 STYLE)
----------------------------------------------------------------------------
GOAL: Crowdsource the discovery of new bypasses to harden the system.

THE MECHANISM:
1. THE ARENA: A local sandbox with "Challenges" (Levels 1-10).
2. HUNTER TOOL: Users run their local agents against the Arena.
3. PROOF OF EXPLOIT (PoE): If an agent succeeds, a bundle is created:
   - Full JSON-RPC conversation logs.
   - Initial system state.
   - Successful bypass payload.
4. SUBMISSION: Hunter submits the bundle to our GitHub/API.
5. VALIDATION: Our "Validator" replays the exploit. If verified, the finding
   is added to the global "Vanguard Signatures" database.

----------------------------------------------------------------------------
4. TECHNICAL ROADMAP
----------------------------------------------------------------------------
PHASE 1: FOUNDATION (POC)
- [ ] Build the stdio Interceptor (Proxy).
- [ ] Implement Layer 1 Static Rules.
- [ ] Log all tool calls to project/audit.log.

PHASE 2: THE GAUNTLET (GAMIFICATION)
- [ ] Create Arena challenges (Level 1: System Prompt Leak).
- [ ] Implement 'vanguard submit' CLI for findings.
- [ ] Build the GitHub Actions Auto-Validator.

PHASE 3: THE INTELLIGENCE LAYER (AI DEFENSE)
- [ ] Integrate local quantized model for Semantic Scoring.
- [ ] Implement 'vanguard update' to pull new signatures.
- [ ] Multi-Model Fallback for the proxy itself.

----------------------------------------------------------------------------
5. SECURITY & ETHICS
----------------------------------------------------------------------------
- ALL challenges must run in a Mock Sandbox (no real system damage).
- Findings are shared openly under an OSS license to protect the ecosystem.
- Designed to align with NIST AI Agent Standards (Feb 2026).
